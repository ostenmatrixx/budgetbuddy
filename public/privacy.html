BudgetBuddy Terms

Privacy Notice

Effective July 23, 2026

BudgetBuddy is a personal-finance application. This notice explains the reference deployment’s data practices. A person or organization operating its own deployment is responsible for publishing contact and jurisdiction-specific terms before accepting users.

Data we process

  • Account email address and authentication metadata.
  • Transactions, notes, categories, budget preferences, and regional settings you enter.
  • Privacy-minimized application error diagnostics when monitoring is enabled.
  • Security and delivery metadata processed by infrastructure providers.

How data is used

Data is used only to authenticate accounts, operate budgeting features, generate requested exports, protect the service, diagnose errors, and satisfy deletion requests. BudgetBuddy does not include advertising and does not sell personal or financial data.

Service providers

The reference architecture uses Supabase for authentication and databases, Vercel for web hosting, Cloudflare Turnstile for abuse prevention, an operator-configured email provider, and optional privacy-minimized Sentry error reporting. Each provider processes the minimum data needed for its role under its own terms.

Retention and backups

Active account data is retained until you delete it or the deployment operator removes it. Account deletion removes the live authentication user and cascades to owned application rows. Encrypted backups may retain deleted data until their scheduled expiration; backup copies are not used for ordinary product access.

Your controls

Account Settings provides fresh JSON and CSV exports, password management, regional preferences, and permanent account deletion. Contact the deployment operator if you cannot access those controls.

Security and children

BudgetBuddy uses encrypted transport, row-level database authorization, abuse protection, and restricted monitoring. No internet service can promise absolute security. The reference deployment is not intended for children under 18.

Questions or security concerns

For the reference repository, use a private GitHub security advisory. Operators must replace this with their own monitored privacy contact before launch.

Review this notice whenever the deployment’s providers or data practices change.
